```sql

SELECT s.estufa_id, e.nome, s.tipo_sensor, s.valor_leitura, s.timestamp
FROM Sensor_Ambiente s
JOIN Estufa e ON s.estufa_id = e.id
WHERE s.tipo_sensor = 'OXIGENIO'
  AND s.valor_leitura < 19.5
  AND s.timestamp >= NOW() - INTERVAL 24 HOUR
ORDER BY s.timestamp DESC;

-- HistГіrico de umidade por estufa (mГ©dia diГЎria)
SELECT estufa_id, DATE(timestamp) AS dia, AVG(valor_leitura) AS media_umidade
FROM Sensor_Ambiente

GROUP BY estufa_id, DATE(timestamp)
ORDER BY dia DESC;

-- Plantas em fase de colheita com estufas ativas
SELECT p.nome_cientifico, p.data_plantio, e.nome AS estufa
FROM Planta p
JOIN Estufa e ON p.estufa_id = e.id
WHERE p.fase_crescimento = 'COLHEITA'
  AND e.status = 'ATIVA';
```

## рҹ”җ MГіdulo 3 вҖ” SeguranГ§a (Spring Security)

### EstratГ©gia de AutenticaГ§ГЈo

O sistema utiliza autenticaГ§ГЈo stateless baseada em **JWT (JSON Web Token)**:

```
1. Cliente вҶ’ POST /auth/login  { email, senha }
2. API valida credenciais no banco (BCrypt.matches)
3. API gera JWT com userId + cargo + expiraГ§ГЈo (8h)
4. Cliente armazena o token
5. Toda requisiГ§ГЈo subsequente: Authorization: Bearer <token>
6. JwtAuthFilter valida o token antes de cada controller
```

### Criptografia de Senhas


### PrГЎticas de SeguranГ§a Implementadas

| PrГЎtica | ImplementaГ§ГЈo | ProteГ§ГЈo |
|---|---|---|
| **Hash de senha** | `BCryptPasswordEncoder` (custo 10) | ExposiГ§ГЈo de credenciais |
| **JWT stateless** | `jjwt` library, expiraГ§ГЈo 8h | SessГөes hijack |
| **Bean Validation** | `@NotNull`, `@Min`, `@Max`, `@Pattern`, `@Email` | Inputs invГЎlidos |

| **SanitizaГ§ГЈo de texto** | RemoГ§ГЈo de tags HTML em campos livres | XSS |
| **ValidaГ§ГЈo de range fГӯsico** | OвӮӮ: 0вҖ“100%, Temp: -273вҖ“500В°C | Payload malicioso |
